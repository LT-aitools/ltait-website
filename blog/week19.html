SVG Logo Animation with Cursor

Inspired by a podcast mention about Trello's animated favicons, we decided to test whether Cursor could animate our company logo.

The Challenge

After hearing about how Trello animates their favicons and how a designer wanted to recreate similar animations after losing original files, we wanted to see if Cursor could animate our SVG logo.

Our Discovery

We initially didn't know that SVGs could be animated with code - we thought you could only make animated GIFs.

Implementation Process

  • Dropped our existing SVG logo directly into Cursor
  • Provided specific requirements: "I want to create an animation whereby the dots are circles, first appear randomly. So not left to right. And then the lines are drawn from top to bottom. I want this animation to last one second total."
  • The first attempt was too fast
  • The second attempt didn't start from scratch (it was already partially populated)
  • Made adjustments and got it working on attempt 3

Results

The animation worked as requested - dots appeared randomly rather than sequentially, followed by lines drawing from top to bottom, completing in one second total.

Grade: A

Quick successful implementation that accomplished exactly what we asked for in just two iterations.

Mid Journey Character Creation and Consistency Testing

We conducted comprehensive testing of Mid Journey's ability to create consistent characters across multiple iterations, uncovering both impressive capabilities and concerning biases.

The Challenge

We wanted to test whether Mid Journey could maintain character consistency - creating the same character in different scenarios while keeping recognizable features.

Our Testing Approach

  • Started with a cartoon cheetah character as our base design
  • Used Mid Journey's character reference feature to maintain consistency
  • Tested different "temperature" settings (400 for exact same character vs. 10 for generally similar)
  • Created multiple scenarios: birthday celebration, different poses, various contexts

Character Consistency Results

  • Excellent consistency: When using character reference at temperature 400, we achieved "exact same individual character" results
  • Spot pattern preservation: For example, Cheeto maintained identical spot patterns across different scenarios
  • Style reference control: We learned to use style reference to maintain flat cartoon aesthetics vs. unwanted 3D rendering
  • Scenario flexibility: Successfully generated the same character in different contexts (birthday party, various poses)

Mascot Creation Process

We also experimented with creating a new LTAIT brand-aligned mascot from scratch
  • Asked ChatGPT to suggest mascot concepts based on our business description
  • Selected a "platypus" concept for being "weird, cute, and adaptable"
  • Experimented with different artistic styles: retro cartoon, steampunk
  • Tested various character attributes and scenarios
  • Used Mid Journey's "animate" button feature on static images

Gender Challenge We Encountered

When trying to create a "female platypus persona," we had difficulty getting clearly female results. Even when specifying "female" and adding details like a lab coat, the results often appeared gender-neutral or masculine. We found that the characters seemed to default to looking more like boys, and creating a clearly female character required more specific prompting.

What Worked Well

  • Character reference system is highly effective for consistency
  • Temperature controls provide good flexibility between exact matches and variations
  • Style reference prevents unwanted artistic interpretation changes
  • Multiple variations per generation give good options to choose from
Grade: A-

Excellent technical capabilities for character consistency and reference systems. Minor points deducted for some difficulty in creating clearly female characters when specified.

Mid Journey Video Generation Capabilities

We tested Mid Journey's animation features to understand how it handles the transition from static images to moving content.

The Challenge

After creating consistent character designs, we wanted to explore Mid Journey's video generation capabilities to bring our characters to life with motion.

Our Testing Process

  • Used an existing video prompt, and prompted Midjourney to use that look/feel but change what the character did (little girl running on the beach, inspired by Netta's child)
  • Also tried creating an animation of a hardware company's logo, turning a pipe into the logo
  • Generated multiple animation variations to compare quality and effectiveness
  • Evaluated the results for smoothness, duration, and overall appeal

Animation Capabilities

  • Simple activation: The animate button provides easy access to video generation
  • Short duration: Animations are limited to approximately 5 seconds
  • Quality output: Generated smooth motion that enhanced the static designs
  • Multiple format options: Could download as GIF or other video formats
  • Variety generation: Like static images, provided multiple animation variations per request

Technical Performance

  • Animations maintained character integrity during motion
  • Smooth transitions without obvious artifacts
  • Appropriate motion that enhanced rather than distracted from the character design
  • Good rendering quality suitable for web or presentation use

Limitations Discovered

  • Duration constraints: 5-second limit restricts storytelling possibilities
  • Limited motion control: Cannot specify exact animation sequences or timing
  • Basic animation types: Motion appears to be generated automatically rather than following detailed direction

Our Assessment

The animations were smooth and maintained character integrity during motion. While limited to 5 seconds and with no detailed motion control, the feature worked well for creating simple animated versions of our characters.

Browser Testing with Claude and MCP Integration

We tackled website testing this week using Claude's new MCP (Model Context Protocol) integration for browser automation.

The Challenge

We needed to test responsive design functionality across different screen sizes for our website's hamburger menu and navigation. :

  • Set up MCP integration with Claude (which was surprisingly straightforward)
  • Asked Claude to create and execute a comprehensive testing strategy
  • The tool automatically generated test plans covering multiple screen sizes and functionality checks
  • It provided detailed reports on what it found during testing

The Process

  • Claude created a structured testing plan without us having to specify the approach
  • It tested responsiveness across tablet and phone screen sizes
  • Performed functionality testing by attempting to click navigation elements
  • Generated detailed reports with pass/fail results and recommendations

What Worked Well

The automation aspect was genuinely impressive. We simply asked "how do I test if the hamburger in the header works in different screen sizes?" and Claude immediately started working, creating test plans and executing them systematically.

The Problem

While the tool worked beautifully from a process perspective, it gave us completely wrong results. It reported that our navigation was working perfectly when we knew it was broken. Even when we manually verified the issues and reported them back, Claude insisted everything was functioning correctly.

Reality Check

We ended up using Cursor to actually fix the navigation issues. When we fed Claude's "everything is working" report back to Cursor, Cursor immediately identified that Claude was wrong and provided the correct diagnosis.

Grade: C

Excellent automation and testing methodology, but fundamentally unreliable results make it impractical for real-world use. The process is impressive, but accuracy is everything in testing.

Email-to-Calendar Automation with Google Apps Script

This project emerged from a real friend's need to automate her workflow around a weekly feminist events newsletter in Israel.

The Business Problem

Our friend receives a weekly newsletter containing multiple event listings (dates, times, locations, descriptions) and wanted them automatically added to her Google Calendar instead of manually processing each email every week.

Our Technical Approach

We used ChatGPT to design the solution, which recommended Google Apps Script as the optimal platform for Gmail-to-Calendar integration.

Implementation Steps

  • Created prompts to help ChatGPT understand the email format and parsing requirements
  • Generated Google Apps Script code to:
  • Connect to Gmail and identify the specific newsletter emails
  • Parse event information using keyword recognition ("invite" and "link" were key identifiers)
  • Create properly formatted calendar entries with event details and relevant links
  • Handle recurring weekly processing automatically

The Development Process

We initially made the mistake of not starting with a proper ChatGPT project, leading to some debugging loops. The key breakthrough came when we:

  • Ran out of credits on Claude (forcing us to switch tools)
  • Created a proper ChatGPT project with full context
  • Provided the latest code version and clear problem description

This allowed ChatGPT to immediately understand where we were stuck and provide working solutions.

Real-World Results

  • Successfully processes newsletter emails automatically
  • Creates calendar entries with proper timing (defaulting to 7 PM when specific times aren't available)
  • Includes both registration links and event description links in calendar entries
  • Saves our friend 15-20 minutes of manual work each week

Future Enhancement Plans

We want to add functionality to click through event links and extract actual event times, rather than defaulting everything to 7 PM. This would require additional web scraping capabilities.

Deployment Strategy

We're planning to share both the script code (for technical users) and a public calendar option (for non-technical users) to maximize accessibility.

Grade: A+

This solved a real problem efficiently with tools that were perfectly suited to the task. The 1 hour 45 minutes of development time created ongoing weekly value, and the Google Apps Script integration was seamless.

Component Library Creation from Screenshots

We explored whether AI could create reusable component libraries from existing website designs, addressing a common challenge in design and development workflows.

The Concept

A designer mentioned that creating prototypes with company-specific branding is challenging with AI tools. We wanted to test if we could extract component libraries from screenshots of existing sites and then use those components to build new designs.

Our Test Setup

  • Used Canva as our test subject (chosen for its distinctive, consistent branding)
  • Took screenshots of various page elements: headers, navigation, hero sections, footers
  • Fed these to Claude with prompts to extract reusable components

The Process

  • Initially, Claude tried to recreate entire pages rather than isolating components
  • After clarifying that we wanted only components, it generated reasonably accurate code representations
  • Successfully identified different UI patterns: buttons, headers, navigation elements, feature sections

What Worked

  • Claude correctly identified and coded many visual elements from the screenshots
  • Generated usable component code that captured Canva's visual style
  • Created organized component libraries with proper categorization

Limitations We Discovered

  • Missing important details like typography specifications (font sizes, weights, etc.)
  • Struggled with contextual elements like dropdown menus that appear on hover
  • Memory limitations caused it to forget the original component library goal when adding new screenshots
  • Spacing and layout issues when attempting to use components to build new pages

Testing Applications

We tried using the component library to build:

  1. Canva Education Hub: A landing page for educators using Canva's existing style
  2. Jerry's Gelato: A completely different business using Canva's design aesthetic

    Results

    Both attempts produced recognizable pages with proper branding elements, but with significant spacing and layout issues that would require manual cleanup.
